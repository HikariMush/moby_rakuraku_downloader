# Release Notes

## v1.1.4

### 新機能

- MP3 出力時のビットレート検証を強化し、無効な値指定を明確にエラーとして扱うようにしました。
- WAV 出力時に `--bitrate` を指定しても無視されるようにし、ユーザーの誤入力による混乱を防止しました。

### 改善点

- `download_track` の音声変換処理を `validate_audio_settings` に分離して、出力設定の整合性を担保しました。

## v1.1.0

### 新機能

- `--format` / `-f` オプションを追加し、出力形式として `mp3` または `wav` を選択可能にしました。
- `--bitrate` / `-b` オプションを追加し、MP3 出力時に `128` / `192` / `256` / `320` kbps を指定可能にしました。
- `320 kbps` の MP3 出力に対応しました。
- トラックごとに元楽曲の形式・ビットレートを検出し、ログで表示するようにしました。
- GUI に出力形式とビットレートの選択 UI を追加しました。

### 改善点

- 既存の MP3 変換処理を汎用化し、出力形式・ビットレートを柔軟に指定できるようにしました。

## v1.1.1

### 新機能

- `CONTRIBUTING.md` と `.github/pull_request_template.md` を追加し、リリース方針と PR チェックリストを明文化しました。
- リリース本文生成フローを GitHub Actions に組み込み、`README_BEGINNER.md` / `README_DEVELOPER.md` へのリンクと追加機能を自動的に含めるようにしました。

### 改善点

- リリース自動化の仕組みを安定化し、今後のタグ付けリリースで方針が徹底されるようにしました。

## v1.1.2

### 新機能

- GitHub Actions のリリース本文生成スクリプトを修正し、タグ付きリリースの本文作成が安定するようにしました。

### 改善点

- リリースワークフローの PowerShell 処理を簡素化して、タグ名マッチングの失敗を回避しました。

## v1.1.3

### 新機能

- Windows の `.exe` 配布版で `ffmpeg` が同梱されていることを README に明記しました。

### 改善点

- 初心者向けドキュメントで `.exe` とソース版の `ffmpeg` 要件を明確化しました。
